*TCP/IP:*
Basicamente es el protocolo de red que se encarga de enviar los datos de un punto a otro, estos se encargan enrutadores, los cuales son los dispositivos encargados de redirigir esos paquetes de un lugar a otro, las IPs son para darles una direccion unica al usuario para que le lleguen esos paquetes al equipo.Funcionan por medio de caracteres de valor total de 32bits, en cada espacio donde tenga un # tiee un valor de 8bits, y estos llegan a las cuatro #, si que por eso puede llegar a 32bits los valors de ls direcciones IPs.Eso era antes, como llegaba a las 4 millones de combinaciones se decidio en cambiar este sistema, la anterior se llamaba IPv4, pero actualmente se usa la IPv6, ya que funcion con 128bits. Los paquetes de datos son paquetes que los mandan las IPs, lo que pasa es que esos paquetes vienen con bits especificos, estos bits contienen informacion para que el servidor las pueda procesar.

Ahora el protoolo TCP es basicamente el que distingue entre si vos sos un cliente o un servidor, el que se asegura de que seas el propietario o el destinario del paquete de datos.

*Los Puertos:*
Los puertos se usan para saber que servicio es el que se esta ocupando, funciona como entrada y salida de datos.

*DNS:*
Los DNS son una forma de poder asignarle un nombre a una direccion ip, para de forma mas facil de identificar de forma humana una pagina o un sitio web, se puede llegar a alquilar.

*DHCP:*
Usado para poder hacerle saber al enrutador en donde te encuentras, es para saber cual es tu direccion ip, cual es la del enrutador, eso lo asigna de forma automatica.

*HTTPS:* 
Es la forma segura y cifrada de navegar por internet, este actua como intermediario entre el cliente y el servidor, lo que pasa es en en plena conexion pueden llegar atacantes que interfieran y roben los paquetes de datos que se esta enviando al servidor, asi que HTTPS es un protocolo que encripta dichos datos con diferentes metodos.

*Status code:*
El status code es un mensaje del resultado de la conexion entre el cliente hacia el servidor, son codigos que significan en ue resulto la conexion, el codigo 200 es un codigo el cual confirma que la conexion salio bien, el codigo 301 indica al nvegador el archivo al que se va a mover, el 404 es para indicar que no se ubico o encontro la web.
